Use it in the
right place

Before

Meeting → Manual note cleanup → Action list → Email summary → Manager review

With Copilot

Meeting → Transcript / Notes → Copilot draft summary + action items → Human review → Final communication

Use Copilot when the task is

✓ Repetitive
✓ Time-consuming but low-risk
✓ Text-heavy
✓ Summarisation or drafting-based
✓ Still subject to human review

Do not use Copilot when the task is

✕ Highly sensitive
✕ Final decision-making
✕ Dependent on confidential inputs
✕ Requiring expert judgment without verification

Action 3 — Good Prompting

Prompting
formula

+ + + +

Click any element to see do's and don'ts

Task — What are you asking Copilot to do?

Do

  • Use action verbs — "Summarise", "Draft", "Compare", "Rewrite"
  • Be task-oriented and reference what you're working on

Don't

  • Use vague commands like "write an email" — Copilot has nothing to work with

Content — What material should it work with?

Do

  • Reference the specific document, transcript, or data in your prompt

Don't

  • Use broad prompts that pull in unnecessary or unrelated data
  • Overload one prompt with multiple unrelated requests

Tone — How should the output sound?

Do

  • State the register explicitly — formal, concise, persuasive
  • Ask for transformation, not generation: "make this more direct"

Don't

  • Leave tone ambiguous — Copilot defaults to generic output

Audience — Who will read the output?

Do

  • Name the reader — senior leadership, a client, your team
  • Specify audience alongside tone and format for coherent output

Don't

  • Assume Copilot knows who the output is for without being told

Format — How should the output be structured?

Do

  • Specify: bullet list, 5-sentence summary, table, or headers
  • Use follow-up prompts to refine — the first output is a starting point

Don't

  • Accept the first output without reviewing or refining it
08

The formula

Task + Context
= results

A strong prompt tells Copilot what to do, what to work with, who will read it, and how to present it. The more specific you are, the more useful the output.

Action 3 — Prompt in Action

See the
difference

Vague prompt

"Write something about our Q2 report."
  • No task orientation — Copilot has to guess
  • No tone, audience, or format specified
  • No reference to actual content
  • Results will be inconsistent every time

Strong prompt

"Summarize the key findings from our Q2 report (attached) for senior leadership. Keep the tone professional and concise. Highlight revenue growth, client onboarding metrics, and risks. Create a 5-sentence summary and a bullet list version."
  • Task-oriented ("Summarize…")
  • References actual content ("Q2 report attached")
  • Tone + audience + output format all specified
  • Asks for transformation, not random generation

Action 4 — Accountability

Manage risks.
Stay in control.

Main risks to watch

  • Reduced critical thinking over time
  • Inaccurate or misleading outputs presented as fact
  • Security and confidentiality exposure
  • Over-dependence reducing individual accountability

How to stay in control

  • Always fact-check — Copilot can be confidently wrong
  • Challenge AI suggestions; don't assume they're correct
  • Maintain independent thinking and manual skills
  • You own the final output — always
10

Core principle

AI assists.
You decide.

Copilot can be confidently wrong. Every output needs a human eye before it goes anywhere. Accountability stays with you — not the tool that helped you draft it.
